# Skill: QA Checklist Generation

Use this skill to create a story-specific QA checklist.

## Inputs

- Spec and acceptance criteria.
- Changed components.
- Known risks.
- Existing tests.
- UI/API/data/cloud impact.

## Output categories

- Functionality
- Style/visual
- Accessibility
- Integration
- Security
- Performance
- Reliability
- Regression

Each item must have expected result, method, status, evidence, and owner if failed.
