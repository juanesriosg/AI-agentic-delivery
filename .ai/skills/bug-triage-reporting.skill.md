# Skill: Bug Triage and Reporting

## Purpose

Turn observed defects into actionable work without derailing unrelated progress.

## Procedure

1. Capture expected vs actual behavior.
2. Reproduce with minimal steps.
3. Capture evidence.
4. Classify severity and impact.
5. Determine whether it blocks the current task.
6. Add or propose a regression test.
7. Create a bug issue or PR note.
8. Continue safe work if the bug is not blocking.

## Bug report format

```md
Bug ID:
Title:
Severity:
Task/PR/Epic:
Expected:
Actual:
Reproduction steps:
Evidence:
Suspected area:
Regression test:
Blocks current task: yes/no
Recommended owner/action:
```
