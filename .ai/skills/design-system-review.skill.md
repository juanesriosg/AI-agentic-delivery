# Skill: Design System Review

Use this skill when UI work touches shared components or repeated patterns.

## Steps

1. Identify whether an existing component should be reused.
2. Review props/API, variants, states, tokens, accessibility, and docs.
3. Avoid one-off styling when a reusable pattern exists.
4. Recommend incremental improvements instead of broad refactors unless approved.
5. Ensure visual QA covers component states.
