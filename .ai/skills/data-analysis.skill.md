# Skill: Delivery Data Analysis

## Purpose

Analyze the engineering work generated by agents to decide whether the system is healthy.

## Analysis questions

- Are agents completing tasks or just opening noisy PRs?
- Are PRs small enough to review?
- Are tests improving?
- Which repos are missing mechanisms?
- Which tasks become blocked most often?
- Are high-risk changes increasing?
- Are agents repeatedly touching protected areas?
- Are manager reviews becoming a bottleneck?
- Are features reaching epic completion?
- Are defects escaping after merge?
